# Horae 記憶畢業到世界書（horae-memory-graduation）

> v0.2.0｜給 Mini 自己用優先｜更新 26/06/24

> v0.2.0 核心修正：不再把可選的 `autoSummaries` 當唯一資料源。即使 Horae「自動摘要」關閉，也會直接讀每樓 `horae_meta.events` 的重要／關鍵事件；另納入永久場景記憶、聚合後的角色／好感度、重要物品與關係資料。active 摘要存在時會取代其覆蓋事件，避免雙寫。

> v0.1.3 修正：升級前殘留的小數、負數與超上限數值會在載入時自動修正，避免意外送出過量 Sub API 請求；每次寫入後會重整整本目標書的 hmg 條目，有關鍵字者只保留全域最新 N 條藍燈／`ignoreBudget`，不再跨批次累積。排序以 Horae 摘要 id 的建立時間為主，舊資料退回覆蓋樓層與 uid。

> v0.1.2 修正：新建世界書改走核心檔名清理、撞名確認與建立流程；重跑以完整 fingerprint 保護內容／標題／關鍵字，舊版無完整 fingerprint 的條目也保守跳過；數值設定會轉整數並依 min/max 限制。v0.1.1 的 Sub API fallback 死條目修復保留。

> 舊版條目若因缺少完整 fingerprint 被跳過，可保留原條目不動；若確定要重新產生，請先在世界書編輯器刪除該條，再重新執行畢業流程。

把 **Horae** 已標記的重要記憶，交給你自選的 **Sub API**（整理用的便宜模型）重寫成通順、好觸發的**世界書條目**，讓很久以前的劇情長存在 AI 眼前——**不要求 Horae 自動摘要、不靠向量、手機友善**。

## 它解決什麼

Horae 每輪都會把「現況」重新貼給 AI，所以現況不太會忘；會忘的是**遠期劇情細節**（第一章那個約定、三十樓前那句話），因為它們寫在聊天正文裡，聊天一長就被酒館從最舊往外丟。世界書條目不在「會被丟」的隊伍裡 → 把舊劇情搬進世界書，AI 就能重新記得。

## 怎麼用（MVP：手動）

1. 在 ST 的**連線設定檔（Connection Profiles）**先存好一個「整理用」的便宜模型（例如 Haiku / Gemini Flash）。
2. 打開本擴充設定面板（擴充設定頁）：
   - 勾「用 Sub API 整理」，選剛剛那個**連線設定檔**。（不想花錢就關掉它＝**省錢模式**：原文照搬，沿用 Horae 的人名／地點等基礎關鍵字。）
   - 選一本**目標世界書**（或按 ＋ 新建一本）。
3. 在一個**有跑 Horae** 的聊天裡，按面板的「**畢業歷史記憶到世界書**」按鈕，或打 `/hmg-graduate`。Horae 自動摘要可以保持關閉。
4. 流程：**選重要事件／永久資料 → Sub API 整理 → 逐條預覽確認 → 寫入世界書**。全程你掌控，不會偷塞。

驗收：到世界書編輯器看條目有沒有出現、`content` 是不是整理後的文字；再跑一次不會重複（同一來源會 upsert 更新，不新增副本）。

## 設計重點（為什麼這樣做）

- **不靠向量**：條目一律 `vectorized=false`，召回靠關鍵字 / 常駐藍燈的純文字掃描，手機零負擔。
- **注入位置 `position=0`（背景區）**：跟 Horae 末尾的「現況」注入錯開，靠後的最新自然優先，避免打架。
- **只讀 Horae、從不寫 Horae**：事件讀各樓 `horae_meta.events`；永久場景與聚合狀態讀 Horae 公開狀態／`chat[0].horae_meta`。`autoSummaries` 只是可選壓縮層。
- **去重**：每條打 `hmg` 標記（同時放 `extensions.hmg` 與頂層欄位雙保險），重跑不重複。
- **藍燈 / 綠燈**：最新 N 條設藍燈常駐，其餘靠 Sub API 或 Horae 基礎關鍵字觸發；沒有關鍵字的條目雖常駐，但不會無上限繞過世界書預算。

## 測試前提

本機制鏈（世界書在 token 預算上排在聊天歷史**之前**先卡位、Horae 注入掛 `CHAT_COMPLETION_PROMPT_READY`）目前**只在 Chat Completion（OpenAI 風格）後端**成立。Text Completion 後端的預算行為未驗，請以 CC 後端測試。

## 除錯

面板的「除錯」按鈕（或 `/hmg-debug`）會把重要事件、摘要與永久資料候選統計印到瀏覽器 Console（F12），並對目標世界書做一次 **save→load 往返測試**。測試條目叫「hmg 往返測試」，可在世界書編輯器手動刪除。

## 解除安裝 / 回退

- 本擴充寫進世界書的條目，標題都以「**時光記憶｜…**」開頭、且帶 `hmg` 標記，可在世界書編輯器手動刪除。
- 停用擴充會自動移除設定面板、解除事件監聽，不殘留。
- 一鍵清除所有 hmg 條目列在 v2 路線圖。

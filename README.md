# Horae 記憶畢業到世界書（horae-memory-graduation）

> v0.1.0（MVP）｜給 Mini 自己用優先｜建立 26/06/22

把 **Horae** 已經壓縮好的「歷史劇情摘要」，交給你自選的 **Sub API**（整理用的便宜模型）重寫成通順、好觸發的**世界書條目**，讓很久以前的劇情長存在 AI 眼前——**不靠向量、手機友善**。

## 它解決什麼

Horae 每輪都會把「現況」重新貼給 AI，所以現況不太會忘；會忘的是**遠期劇情細節**（第一章那個約定、三十樓前那句話），因為它們寫在聊天正文裡，聊天一長就被酒館從最舊往外丟。世界書條目不在「會被丟」的隊伍裡 → 把舊劇情搬進世界書，AI 就能重新記得。

## 怎麼用（MVP：手動）

1. 在 ST 的**連線設定檔（Connection Profiles）**先存好一個「整理用」的便宜模型（例如 Haiku / Gemini Flash）。
2. 打開本擴充設定面板（擴充設定頁）：
   - 勾「用 Sub API 整理」，選剛剛那個**連線設定檔**。（不想花錢就關掉它＝**省錢模式**：原文照搬、無關鍵字。）
   - 選一本**目標世界書**（或按 ＋ 新建一本）。
3. 在一個**有跑 Horae、且夠長**的聊天裡，按面板的「**畢業歷史記憶到世界書**」按鈕，或打 `/hmg-graduate`。
4. 流程：**選要畢業的摘要 → Sub API 整理 → 逐條預覽確認 → 寫入世界書**。全程你掌控，不會偷塞。

驗收：到世界書編輯器看條目有沒有出現、`content` 是不是整理後的文字；再跑一次不會重複（同一條摘要會 upsert 覆蓋，不新增）。

## 設計重點（為什麼這樣做）

- **不靠向量**：條目一律 `vectorized=false`，召回靠關鍵字 / 常駐藍燈的純文字掃描，手機零負擔。
- **注入位置 `position=0`（背景區）**：跟 Horae 末尾的「現況」注入錯開，靠後的最新自然優先，避免打架。
- **只讀 Horae、從不寫 Horae**：只讀 `chat[0].horae_meta.autoSummaries`，自己的狀態寫獨立世界書。
- **去重**：每條打 `hmg` 標記（同時放 `extensions.hmg` 與頂層欄位雙保險），重跑不重複。
- **藍燈 / 綠燈**：最新 N 條設藍燈常駐（保證在場）、其餘靠 Sub API 抽的關鍵字綠燈觸發；省錢模式因為沒有關鍵字，會全部設藍燈。

## 測試前提

本機制鏈（世界書在 token 預算上排在聊天歷史**之前**先卡位、Horae 注入掛 `CHAT_COMPLETION_PROMPT_READY`）目前**只在 Chat Completion（OpenAI 風格）後端**成立。Text Completion 後端的預算行為未驗，請以 CC 後端測試。

## 除錯

面板的「除錯」按鈕（或 `/hmg-debug`）會把最新的 `autoSummaries` 印到瀏覽器 Console（F12），並對目標世界書做一次 **save→load 往返測試**，回報 `extensions.hmg` 與頂層標記哪一種在你的環境存活（去重就靠它）。測試條目叫「hmg 往返測試」，可在世界書編輯器手動刪除。

## 解除安裝 / 回退

- 本擴充寫進世界書的條目，標題都以「**時光記憶｜…**」開頭、且帶 `hmg` 標記，可在世界書編輯器手動刪除。
- 停用擴充會自動移除設定面板、解除事件監聽，不殘留。
- 一鍵清除所有 hmg 條目列在 v2 路線圖。
